# Day 4 - Printing Department

> Written in Rust

This is a mapping search problem which works on a 2D map. The map contains a series of nodes in a
2-dimensional space and we need to know how many nodes have 3 or fewer direct neighbours (i.e.
directly or diagonally adjacent).

The following can be said about our puzzle input:

* All lines will be of equal length
* Lines can only contain the characters `^[.@]+$`

For example, given the input:

```text
..@@.@@@@.
@@@.@.@.@@
@@@@@.@.@@
@.@@@@..@.
@@.@@@@.@@
.@@@@@@@.@
.@.@.@.@@@
@.@@@.@@@@
.@@@@@@@@.
@.@.@@@.@.
```

We can identify the following nodes as having 3 or fewer direct neighbours:

```text
..xx.xx@x.
x@@.@.@.@@
@@@@@.x.@@
@.@@@@..@.
x@.@@@@.@x
.@@@@@@@.@
.@.@.@.@@@
x.@@@.@@@@
.@@@@@@@@.
x.x.@@@.x.
```

Meaning we have a total of `13` chosen nodes.

## This script

This script uses Rust and can be run with the following command:

```shell
cargo run -- -i input.txt
```

To enable part two, please include the `--enable-loop` flag. I.e.:

```shell
cargo run -- -i input.txt --enable-loop
```

Debug logging can be enabled with `RUST_LOG=debug` as a command prefix.

A test suite has been provided with this library, and can be run with the following:

```shell
cargo test
```
